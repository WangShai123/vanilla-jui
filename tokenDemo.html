vanilla-jui design tokens

让背景更中性，色彩更容易落到真实界面里。

这个页面把基础背景、固定主题色和语义状态色拆开展示，方便你直接判断每个 token 在 light / dark 两种模式下更适合放到哪里。底层背景已经从偏深蓝的冷感，调整成更通用的深灰中性基底。

当前模式: Light

当前模式下的层级示意

最适合的落点

  • 背景层用中性灰，不抢色彩主题。
  • 主题色用于按钮、链接、聚焦和品牌识别。
  • 状态色用于警告、成功、信息与破坏性操作。

UI 结构

这组 token 负责页面骨架、表面层级、边框、遮罩和文本主次关系。

固定主题色

这 15 个颜色适合做主题皮肤、标签、统计图、分类卡片和品牌识别，不必强行都当作背景。

语义状态色

状态色更适合告诉用户“发生了什么”，例如成功、提醒、危险、信息提示。

推荐的应用方式

下面三个小样例把 background、surface、tone 和 state 放在一起，展示更接近真实页面的效果。

主操作区

适合：主按钮、核心 CTA、关键导航。

用 --tone-solid 做强对比，搭配 --tone-fg 保证可读性，外层用 --ui-surface-raised 让按钮更像可点击的动作层。
